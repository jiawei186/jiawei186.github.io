Relational-algebra

关系代数是一种程序查询语言
关系代数使用对关系的运算来表达查询

运算符

关系代数中的运算符分为传统的集合运算符, 专门的关系运算符

运算符分类 符号 解释
传统的集合运算符 ∪\cup∪ 并
−-− 差
∩\cap∩ 交
×\times× 笛卡尔积
专门的关系运算符 σ\sigmaσ 选择
∏\prod∏ 投影
⋈\Join⋈ 连接
÷\div÷ 除

集合运算只涉及到行
专门的关系运算符同时涉及到行和列

专门的关系运算符还需要比较运算符和逻辑运算符的辅助

并, 差, 笛卡尔积, 选择, 投影这5种运算是关系代数的基本运算
交, 连接, 除可以用5种基本运算来表达, 这3种运算并没有增加关系代数的能力, 但简化了表达

以上8种运算, 经有限次复合形成的表达式被称为关系代数表达式

传统的集合运算

设关系R和关系S具有相同的目n, 且相同的属性取自同一个域
t是元组变量, t∈Rt\in Rt∈R表示t是R的一个元组

并

R与S的并
R∪S={t∣t∈R∨t∈S}R\cup S = \{ t| t\in R \lor t \in S\}R∪S={t∣t∈R∨t∈S}

差

R与S的差
R−S={t∣t∈R∨t∉S}R - S = \{ t| t\in R \lor t \notin S \}R−S={t∣t∈R∨t∈/​S}

交

R与S的交
R∩S={t∣t∈R∧t∈S}R\cap S = \{ t| t\in R \land t\in S \}R∩S={t∣t∈R∧t∈S}

R与S的交可以转化为差
R∩S=R−(R−S)R\cap S = R - (R - S)R∩S=R−(R−S)

笛卡尔积

关系代数里的笛卡尔积应该是广义的笛卡尔积, 因为这里的元素是元组

n目关系R和m目关系S的笛卡尔积是一个(n+m)列的元组的集合
结果的一个元组的前n列是关系R的一个元组, 后m列是关系S的一个元组

若R有k1个元组, S有k2个元组, 则R和S的笛卡尔积有k1×k2k_{1}\times k_{2}k1​×k2​个元组

R和S的笛卡尔积
R×S={trts⏠∣tr∈R∧ts∈S}R\times S = \{ \overgroup{t_{r}t_{s}}| t_{r}\in R \land t_{s}\in S\}R×S={tr​ts​​∣tr​∈R∧ts​∈S}

专门的关系运算

关系模式为R(A1,A2,...,An)R(A_{1}, A_{2}, ..., A_{n})R(A1​,A2​,...,An​), 它的一个关系设为R
t∈Rt\in Rt∈R表示t是R的一个元组
t[Ai]t[A_{i}]t[Ai​]表示元组t中相应于属性AiA_{i}Ai​的一个分量

Ai1,Ai2,...,AikA_{i1}, A_{i2}, ..., A_{ik}Ai1​,Ai2​,...,Aik​是A1,A2,...,AnA_{1}, A_{2}, ..., A_{n}A1​,A2​,...,An​中的一部分
则称A=Ai1,Ai2,...,AikA=A_{i1}, A_{i2}, ..., A_{ik}A=Ai1​,Ai2​,...,Aik​为属性列或属性组
t[A]=(t[Ai1],t[Ai2],...,t[Aik])t[A]=(t[A_{i1}], t[A_{i2}], ..., t[A_{ik}])t[A]=(t[Ai1​],t[Ai2​],...,t[Aik​])表示元组t在属性列A上诸分量的集合
A‾\overline{A}A则表示Ai1,Ai2,...,AikA_{i1}, A_{i2}, ..., A_{ik}Ai1​,Ai2​,...,Aik​中去掉A1,A2,...,AnA_{1}, A_{2}, ..., A_{n}A1​,A2​,...,An​后剩余的属性组

R是n目关系, S是m目关系
tr∈R,ts∈S,trts⏠t_{r}\in R, t_{s}\in S, \overgroup{t_{r}t_{s}}tr​∈R,ts​∈S,tr​ts​​被称为元组的连接或元组的串接
trts⏠\overgroup{t_{r}t_{s}}tr​ts​​是一个n+m列的元组, 它的前n个分量为R中的一个n元组, 后m个分量为S中的一个m元组

一个关系R(X, Z), X和Z为属性组, X和Z共同组成了R
当t[X]=x时, x在R中的象集为Zx={t[Z]∣t∈R,t[X]=x}Z_{x}= \{ t[Z] | t\in R, t[X] = x \}Zx​={t[Z]∣t∈R,t[X]=x}
x在R中的象集为属性组X上值为x的诸元组在Z上分量的集合

选择, selection, 限制, restriction

选择是挑选在关系R中满足给定条件的元组
σF(R)={t∣t∈R∨F(t)=′真′}\sigma_{F}(R) = \{ t| t\in R \lor F(t) = '真' \}σF​(R)={t∣t∈R∨F(t)=′真′}
F表示选择条件, 它是一个逻辑表达式, 去逻辑值真或假
F的基本形式为X1θY1X_{1}\theta Y_{1}X1​θY1​

分类 θ\thetaθ表示比较运算符 解释
比较运算符 <<< 小于
⩽\leqslant⩽ 小于等于
=== 等于
<><><> 不等于
⩾\geqslant⩾ 大于等于
>>> 大于
逻辑运算符 ¬\lnot¬ 非
∧\land∧ 与
∨\lor∨ 或

投影, projection

投影是从关系R中选择出若干属性列来组成新的关系
$ \prod_{A}® = {t[A]| t\in R }$
A为R中的属性组

一个关系投影之后, 如果有些元组的A属性完全相同, 那它们可能会被消减为一行元组

连接, join, θ\thetaθ连接

连接是从两个关系的笛卡尔积中选取属性间满足一定条件的元组
R⋈AθBS={tr⏠ts∣tr∈R∧ts∈S∧tr[A]θts[B]}R \underset{A\theta B}{\Join} S = \{ \overgroup{t_{r}}{t_{s}}| t_{r}\in R \land t_{s}\in S \land t_{r}[A] \theta t_{s}[B] \}RAθB⋈​S={tr​​ts​∣tr​∈R∧ts​∈S∧tr​[A]θts​[B]}
A和B分别是R和S上列数相等且可比的属性组, θ\thetaθ是比较运算符
连接运算从R和S的笛卡尔积R×SR\times SR×S中选取R关系在A属性组上的值与S关系在B属性组上的值满足比较关系θ\thetaθ的元组

等值连接, equijoin
θ\thetaθ为"="的连接运算为等值连接
R⋈A=BS={tr⏠ts∣tr∈R∧ts∈S∧tr[A]θts[B]}R \underset{A= B}{\Join} S = \{ \overgroup{t_{r}}{t_{s}}| t_{r}\in R \land t_{s}\in S \land t_{r}[A] \theta t_{s}[B] \}RA=B⋈​S={tr​​ts​∣tr​∈R∧ts​∈S∧tr​[A]θts​[B]}

自然连接, natural join
连接中, 进行比较的分量是同域集同名的属性组, 并且在结果中把重复的属性组消减, 这就是自然连接
R⋈S={tr⏠ts[U−A]∣tr∈R∧ts∈S∧tr[A]θts[A]}R \Join S = \{ \overgroup{t_{r}}{t_{s}}[U-A]| t_{r}\in R \land t_{s}\in S \land t_{r}[A] \theta t_{s}[A] \}R⋈S={tr​​ts​[U−A]∣tr​∈R∧ts​∈S∧tr​[A]θts​[A]}
R和S中具有相同的属性组A, U为R和S的全体属性集合

R中的属性组的值对应S中的属性组的值时, 可能会出现R中的属性组的元组的一个值只在R中出现而S中没有的情况, 这些元组被称为R的悬浮元组
默认的自然连接是把这些元组舍弃掉的

外连接 左外连接 右外连接
同时保留两个关系的悬浮元组 只保留左边关系R的悬浮元组 只保留右边关系S的悬浮元组

除, division

关系R(X, Y)和关系S(Y, Z), X和Y和Z都是属性组, R中的Y和S中的Y是同域集同名的属性组
R除S得到P, P中的元组是X中的满足以下条件的元组
元组在X上分量值x的象集Yx包含S在Y上投影的集合
R÷S={tr[X]∣tr∈R∧∏Y(S)⊆Yx}R\div S= \{t_{r} [X]| t_{r}\in R \land \prod_{Y}(S) \sube Y_{x} \}R÷S={tr​[X]∣tr​∈R∧∏Y​(S)⊆Yx​}
Yx是x在R中的象集, x=tr[X]
